### En:
#### Task 1
What You Need to Do
You need to implement a technique commonly used on e-commerce websites to attract customer attention to promotions.
Create a popup notification window offering a gift to site visitors.
The gift should be selected randomly from an array of objects provided below.
Based on the selected object, a popup card should be generated.
The card should look similar to the one shown in the image below.
It must appear three seconds after the site loads.
Please note that the card includes a button labeled “Great!”.
When this button is clicked, the offer card should be closed (removed).
Pixel-perfect layout is not required for this task, but the overall style and appearance of the card should be preserved.

Recommendation
To select a random object from the array, you can use a random number generation algorithm.
A timer will help delay the creation of the card.

Task Completion Criteria
- No errors in the console
- The web application code works correctly and meets the task requirements
- A card with a random offer appears three seconds after the page loads in the browser
- DOM elements are styled according to the provided layout

### Ru:
#### Задача 1
Что нужно сделать
Вам нужно реализовать приём, который часто используется на сайтах интернет-магазинов, чтобы привлечь внимание покупателя к акции. Создайте  всплывающее попап-окно уведомления о подарке для посетителей сайта. Подарок должен выбираться случайным образом из массива объектов, который дан ниже.

На основе выбранного объекта должна создаваться всплывающая карточка. Карточка должна выглядеть как на изображении ниже. Её нужно показывать через три секунды после запуска сайта. Обратите внимание, что у карточки есть кнопка «Отлично!». При клике на эту кнопку карточка предложения должна быть закрыта (удалена).

В этом задании не требуется вёрстка pixel perfect, но общая стилистика и вид карточки должны быть сохранены.

Рекомендация
Для выбора произвольного объекта в массиве вы можете использовать алгоритм создания случайного числа. А таймер поможет отложить создание карточки.

Критерии выполнения задания
В консоли нет ошибок.
Код веб-приложения работает корректно, в соответствии с заданием.
Карточка с произвольным предложением появляется через три секунды после запуска страницы в браузере.
DOM-элементы стилизованы в соответствии с предложенным макетом.
